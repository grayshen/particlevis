File Threading Notes

Whenever a statefile or a particle descriptor is loaded, ParticleVis will create a background I/O thread to handle the retrieval and parsing of these files. Only a single statefile loading thread and a single particle descriptor thread can run concurrently with ParticleVis. As the progress of file loading continues the data will become immediately available for visualization.

To interrupt either of these processes, select "Terminate All File Processes" from the File menu. All I/O threads will immediately be halted. To continue loading the file you must reselect it using the appropriate File menu command. If you wish to load a different statefile or XML file while another is loading, it is necessary to select this command first.

To maximize the speed of file loading, it is possible to disable visualization entirely until the file is completed by selecting "Suspend Drawing Processes" from the File menu. No rendering will occur while this option is enabled, but the control dialog will reflect the number of frames loaded. Simply disable the option when you wish to visualize the data again. This option is also useful if you wish to keep ParticleVis in the background without it consuming CPU cycles.
