WaveFront OBJ File

The WaveFront OBJ mesh files are arranged in a list format with three categories:

  • Vertices (v)- 3D Points
  • Vertex normals (vn)- Define lighting angles for polygonal smoothing
  • Texture Vertices (vt)- Define texture coordinates
  • Faces (f)- Defined by vertices, vertex normals and texture vertices
The order of listing implicitly defines the index of the element. For example, the first vertex listed is vertex 1.
Vertex normals correlate to vertex indices implicitly by position in the list, eg. the first vertex normal represents the first vertex.
Faces are defined by vertex indices. (See "Syntax" below for details)

Syntax

Vertex syntax:
v [x] [y] [z]

Texture Vertex
vt [u] [v] [w]
where [u] is the horizontal direction, [v] is the vertical direction and [w] is the texture depth.
[v] and [u] are optional arguments.

Vertex Normal Syntax:
vn [x] [y] [z]
where [x] [y] [z] form a unit vector.

Face Syntax:
f [v1]/[vt1]/[vn1] [v2]/[vt2]/[vn2] [v3]/[vt3]/[vn3] ...
This defines a face between the vertices [v1]-[v...] with texture vertices [vt1]-[vt...] and normals [vn1]-[vn...]
[vn] and [vt] are optional arguments: f [v1] [v2] [v3] [v...] is valid.

Other:
You may comment lines by starting with a #.
You may specify geometry grouping:
g [groupname]
where [groupname] is a no-whitespace string. ParticleVis does not employ this information.

Example

The following is an example which defines a triangular face.
# Defines a triangle in 3-Space without meaningful normals
# Texture vertices are defined but not used.
g My_triangle
v 0.000 0.000 0.000
v 10.234 5.543 -6.001
v 1.546 6.554 3.222
vt 1 2
vt 1 -1
vt 0 1
vn 0 0 1
vn 0 0 1
vn 0 0 1
f 1//1 2//2 3//3
See http://www.robthebloke.org/source/obj.html for more information and .OBJ capability.
