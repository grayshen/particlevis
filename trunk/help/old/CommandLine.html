Command Line Options

It is possible to pass ParticleVis command-line options to modify its startup parameters, and also invoke background-rendering functionality.

-f filename
Specify an input state file name that is loaded on startup.
-bf filename
Specify input state file name for a background rendering job.
-d filename
Specify XML particle descriptor file to be loaded on startup.
-tex filename
Specify a bitmap to load for textures on startup.
-nolights
Disable lighting (on by default) at startup.
-o filename
Specify output filename for image dumps.
-x n
Specify width of render-window in pixels.
-y n
Specify height of render-window in pixels.
-maxframes n
Specify max number of frames to render in background mode.
-pbuffer
Use pbuffers to render in background mode, allowing hardware acceleration. This option is highly recommended if your graphics card supports it.
-noascii
Disable dump of ASCII data when saving frames.
-cam xyz xyz
Force camera to point from 1st xyz (camera position) to 2nd xyz (lookat point).
-cone xyz xyz a d
Render a translucent cone pointing from xyz to xyz, angle = a, intersecting with a cylinder of radius = d.
-color n
Specify false-color coloring scheme using: 1 = translational velocity, 2 = angular velocity.
-vector n
Switch on vectors of type: 1 = translational, 2 = angular, 3 = normals. You may specify -vector multiple times to switch on different vectors.
-stereo
Enable stereo output for video cards that support it. This is currently an experimental feature.
-?
Display a help dialog for these command line options.

Whenever background rendering is enabled (with -bf) the application will attempt to load and render with all specified options in a background task. No user input beyond the initial command is necessary.
